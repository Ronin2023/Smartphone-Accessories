## How to Commit

### Option 1: Single Commit (Recommended)

```powershell
# Stage all changes
git add .

# Commit with message from file
git commit -F Documentations/COMMIT-MESSAGE.txt

# Or copy-paste the message above
git commit -m "feat: Implement clean URLs without .php extensions" -m "See full details in commit description"

# Push to GitHub
git push origin main
```

### Option 2: Staged Commits (Detailed)

```powershell
# 1. Commit configuration changes
git add .htaccess .htaccess.production
git commit -m "feat: Add clean URL rewrite rules and production config"

# 2. Commit main page updates
git add index.php contact.php products.php compare.php about.php check-response.php
git commit -m "refactor: Update main pages to use clean URLs"

# 3. Commit admin panel updates
git add admin/*.php
git commit -m "refactor: Update admin panel navigation to use clean URLs"

# 4. Commit utility files
git add user_dashboard.php maintenance*.php connection-error.php rate-limit-checker.php
git commit -m "refactor: Update utility pages to use clean URLs"

# 5. Commit JavaScript updates
git add js/main.js js/products.js
git commit -m "refactor: Update JavaScript to use clean URLs"

# 6. Commit testing scripts
git add test-clean-urls.ps1 start-ngrok-testing.ps1
git commit -m "test: Add comprehensive clean URLs testing scripts"

# 7. Commit documentation
git add Documentations/*.md
git commit -m "docs: Add comprehensive clean URLs documentation"

# 8. Push all commits
git push origin main
```

### Option 3: Feature Branch (Best Practice)

```powershell
# Create feature branch
git checkout -b feature/clean-urls

# Stage and commit all changes
git add .
git commit -m "feat: Implement clean URLs without .php extensions"

# Push feature branch
git push -u origin feature/clean-urls

# Later: Merge to main
git checkout main
git merge feature/clean-urls
git push origin main

# Delete feature branch
git branch -d feature/clean-urls
git push origin --delete feature/clean-urls
```

## Commit Verification

```powershell
# Check what will be committed
git status

# See detailed changes
git diff

# See staged changes
git diff --staged

# See commit history
git log --oneline -5

# See files in last commit
git show --name-status
```

## Tag Release (Optional)

```powershell
# Create version tag
git tag -a v2.0.0 -m "Clean URLs Implementation"

# Push tag to GitHub
git push origin v2.0.0

# List tags
git tag -l
```

## GitHub Actions (After Push)

Your push will trigger any GitHub Actions workflows if configured.

## Rollback (If Needed)

```powershell
# Revert last commit (keep changes)
git reset --soft HEAD~1

# Revert last commit (discard changes)
git reset --hard HEAD~1

# Revert specific file
git checkout HEAD -- .htaccess
```
